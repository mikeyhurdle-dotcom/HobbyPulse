## ⚠️ Two real bugs blocking growth

These are the only two things I'd interrupt anything for. Both are 1-PR fixes — the second one I can do for you right now.

### 1. TabletopWatch /blog renders no posts despite 6 posts on disk
- Phase 2 renamed the runtime slug `warhammer` → `tabletop`.
- `lib/blog.ts` reads `content/blog/{slug}/*.md`.
- The directory is still `content/blog/warhammer/` (6 posts) — no `content/blog/tabletop/`.
- Result: production `/blog` says "No posts" even though 6 exist.
- **Fix**: rename `content/blog/warhammer/` → `content/blog/tabletop/`. One git mv + push. I can ship this in a single commit if you greenlight. SimRaceWatch is unaffected (its slug is still `simracing`, directory is `content/blog/simracing/`).

### 2. Board games directory is empty so /boardgames/games/<slug> pages don't render
- `board_games` table has 0 rows.
- This blocks: the Phase 5 price-comparison widget (which is wired correctly — just no data), the `/boardgames/games/*` detail pages, the `/boardgames/recommend` quiz endpoint.
- The seed path was the BGG XML API. **Update 2026-04-27 16:00**: BGG sunset their XML API entirely. New JSON API at `api.geekdo.com/api/geekitems` works but takes one ID at a time (vs old `/xmlapi2/thing?id=1,2,3,...` which took 20). Whole different field shape too — every name in `lib/bgg.ts` would change. This is a 60–90 minute careful migration job, not a one-shot run. **I overestimated this in v1 of this doc.**
- **Status**: schema is now aligned (migration 017 applied), but the import client needs a real rewrite. Parked until someone has the time to do it properly.
- **Workaround in flight**: the price-comparison widget gracefully renders nothing when the join finds no matching listings, so the rest of the site is unaffected. `/boardgames/games/<slug>` 404s for now (no rows = no static params generated = page doesn't exist). Site visitors don't see a broken page; they see no game directory.
